# 养鸟桌宠游戏 · 最简可行版（MVP）项目需求文档

> 版本：v1.0
> 目标：使用godot用最短时间（约 6-8 天）验证“识图出鸟”玩法的趣味性，并产出可运行的桌面宠物原型。

------

## 1. 项目概述

**一句话描述**：
一款 Windows 桌面宠物游戏，用户点击按钮选择任意图片，程序识别图片中的环境类型（森林、沙漠、海洋、城市、雪山），并在桌面上显示一只与环境对应的预设计鸟类。

**核心价值**：
将“选图→识图→出鸟”的惊喜感，转化为轻量、治愈的桌面陪伴体验。

**目标用户**：
喜欢桌面宠物、收集元素、休闲治愈类游戏的 PC 用户（尤其是对鸟类有兴趣的玩家）。

**成功标准（MVP阶段）**：

- 用户能成功选择一张本地图片
- 程序能返回一个环境标签（5种之一）
- 桌面上出现对应环境的鸟（共5种基本鸟种）
- 鸟能在桌面底部左右走动，有待机动画
- 用户能右键退出游戏

------

## 2. 功能需求（按优先级排序）

### P0（必须实现，否则不可用）

| ID   | 功能                 | 描述                                                         |
| ---- | -------------------- | ------------------------------------------------------------ |
| F1   | 桌面透明置顶窗口     | 程序窗口无边框、透明背景、置顶、可拖拽，鸟显示在桌面上方     |
| F2   | 基础鸟动画           | 鸟有“待机”（身体呼吸浮动）和“移动”（左右平移）两种状态       |
| F3   | 图片选择器           | 用户点击按钮可打开系统文件对话框，选择图片（jpg/png）        |
| F4   | 环境识别（本地部署） | 用TensorFlow Lite部署一个极小的图像分类模型（比如`EfficientNet-Lite0`，4MB左右），只分类5-7个环境类别，CPU实时推断毫秒级。 |
| F5   | 环境→鸟种映射        | 根据识别结果，显示对应的鸟（更换精灵/颜色/配件）             |
| F6   | 退出机制             | 右键点击鸟（或系统托盘图标）显示菜单，可选择“退出”           |

### P1（增强体验，但不阻塞核心流程）

| ID   | 功能            | 描述                                                         |
| ---- | --------------- | ------------------------------------------------------------ |
| F7   | 简单的鼠标交互  | 鼠标划过鸟时，鸟会转头看鼠标（可选）或发出短促叫声（可选）   |
| F8   | 空闲自动待机    | 用户无操作1分钟后，鸟进入“瞌睡”状态（动作变慢或闭眼）        |
| F9   | 内置5种基础鸟种 | 森林鸟（绿色/短喙）、沙漠鸟（棕色/长喙）、海洋鸟（蓝白色/翅膀细长）、城市鸟（灰色/不怕人）、雪山鸟（白色/圆胖） |

## 3. 非功能需求

| 类型     | 要求                                     |
| -------- | ---------------------------------------- |
| 平台     | Windows 10/11（64位）                    |
| 性能     | CPU占用 < 5%（空闲时），内存 < 150MB     |
| 启动速度 | 从双击exe到鸟出现在桌面 ≤ 3秒            |
| 识别时间 | 用户选择图片到鸟切换 ≤ 5秒（含网络请求） |
| 隐私     | 图片仅上传至识别API，不保存任何用户图片  |
| 稳定性   | 连续运行8小时不崩溃，不泄漏内存          |
| 安装     | 绿色免安装，单个exe文件或解压即用        |

## 4. 技术栈建议（Godot路线）

| 模块     | 选型                                | 理由                               |
| -------- | ----------------------------------- | ---------------------------------- |
| 游戏引擎 | Godot 4.x                           | 开源、轻量、2D性能好、透明窗口简单 |
| 语言     | GDScript                            | 与Python类似，快速开发             |
| 透明窗口 | Godot内置设置 + `DisplayServer`     | 无需插件即可实现无边框透明置顶     |
| 图片选择 | `DisplayServer.file_dialog_show()`  | 原生对话框，无额外依赖             |
| 环境识别 | TensorFlow Lite                     | 轻量化本地化                       |
| 动画     | `AnimationPlayer` + 程序化位移      | 无需逐帧绘制，只用缩放/旋转        |
| 美术     | 极简几何风格，纯代码绘制或极少量png | 省去大量美术时间                   |
| 数据存储 | JSON文件（物种映射表）              | 简单易懂                           |

## 5. 数据设计（最小集）

### 环境标签映射表（硬编码）

text

```
forest   → 森林鸟（绿、短喙、翅膀圆）
desert   → 沙漠鸟（棕、长喙、冠羽）
ocean    → 海洋鸟（蓝白、尖喙、翼展长）
city     → 城市鸟（灰、短喙、不怕人）
snow     → 雪山鸟（白、圆胖、无冠羽）
```

### 识别API输出示例（以Clarifai为例）

输入图片后，API返回的标签可能是 `"tree"`, `"woodland"` → 映射为 `forest`；
`"sand"`, `"dune"` → 映射为 `desert`。
需要写一个简单的转换函数。

## 6. 美术资源清单（极简）

### 用纯代码绘图方式（推荐MVP）

- 用 `draw_circle()`, `draw_polygon()`, `draw_line()` 等函数在 `_draw()` 中绘制鸟的身体、眼睛、喙、冠羽、翅膀。
- 每种鸟只需改变颜色和几个几何参数（喙长、冠羽数量等）。
- **零图片资产**，程序体积极小。

### 若偏好简单图片

制作5张 64x64 的透明背景PNG：

- 每种鸟一个完整精灵图（待机与移动共用同一张图，通过位移模拟走路）。
  可使用 Kenney 或 OpenGameArt 的基础圆形鸟类，稍作修改。

------

## 7. 用户界面草图（极简）

- **主界面**：无窗口标题栏，只有一个透明区域显示鸟。
- **按钮**：一个小齿轮图标或“+”按钮，悬浮在鸟旁边（或右键菜单中调用）。
- **右键菜单**：
  - 选择图片（识图）
  - 退出

**不需要任何复杂的设置界面。**
